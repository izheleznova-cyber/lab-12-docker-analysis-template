# 🐳 Ура 12: Анализ логов в Docker
> «У меня работает» → «Работает везде»

## 🎯 Цель
Упаковать скрипт `analysis.py` в контейнер, чтобы он запускался одной командой на любом компьютере.

## 📦 Что внутри
- `Dockerfile` — рецепт образа
- `docker-compose.yml` — кнопка «запустить всё»
- `analysis.py` — ваш код анализа
- `game.py` — игра для генерации логов
- `verify.sh` — самопроверка

---

## 🚀 Пошаговое выполнение

### 🔹 Шаг 1: Проверка Docker

Docker уже установлен на вашем компьютере. Нужно убедиться, что он работает:

**Windows/Mac:** 
- Иконка с китом 🐳 должна быть в трее (возле часов) и на панели задач

**Ubuntu:**
```bash
docker --version
docker compose version

Если команды показали версии — всё готово, переходим дальше.

⚠️ Если видите ошибки:
«Cannot connect to the Docker daemon» — запустите Docker Desktop
«command not found» — обратитесь к преподавателю

🔹 Шаг 2: Миссия «Диагональ» — получаем логи (Локально)
Важно: Игру запускаем на своём компьютере (не в Docker!), так как ей нужно графическое окно.
Запустите игру:
python3 game.py

Пройдите миссию по кратчайшей траектории от А к В и обратно (по-возможности по диагонали)
Проверьте что после завершении миссии в репозитории на компьютере появился файл с логами.

🔹 Шаг 3: Анализ в Docker (5 минут)
Теперь запускаем контейнер для анализа:
docker compose up

Что происходит:
Docker собирает образ из Dockerfile (первый раз ~2-3 минуты)
Запускает контейнер с вашим analysis.py
Скрипт находит файл log_*.json, строит график
Сохраняет trajectory_*.png в папку проекта

Проверка результата:
Откройте файл trajectory_*.png
✅ Зачёт: График показывает траекторию близкую к диагонали
❌ Переделка: График кривой или повторяет старые данные


🔹 Шаг 4: Отправка в Git
git add .
git commit -m "feat: diagonal trajectory mission accomplished"
git push origin main

 Критерии сдачи
Dockerfile собирается без ошибок
.dockerignore исключает venv, __pycache__, картинки
docker compose up запускает анализ и сохраняет график
График показывает диагональную траекторию (прямая линия)
В репозитории есть: log_name.json, trajectory_name.png, analysis.py
Коммиты чистые (без fix1, ok, asdf)
